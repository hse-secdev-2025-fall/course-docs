# Чек‑лист перед отправкой PR (и для само‑проверки)

## Перед открытием PR
- [ ] Ветка названа как `pXX-*` (пример: `p03-nfr`)
- [ ] Описание по шаблону: **контекст → что сделано → как тестировал(а) → риски**
- [ ] Нет секретов/мусора в репозитории
- [ ] Обновлён `README`/конфиги/миграции (если нужно)

## Качество/безопасность
- [ ] Линтеры/форматирование зелёные (`ruff`, `black`, `isort`)
- [ ] Тесты (`pytest -q`) проходят локально
- [ ] Негативные/граничные кейсы покрыты (если применимо)
- [ ] Структурные ошибки возвращаются корректно
- [ ] CORS/заголовки/конфиги - безопасные значения (если изменяли)

## Процесс
- [ ] Запрошено ревью у `@hse-secdev-2025-fall/instructors`
- [ ] Ответы на комментарии ревью: **blocking** закрыты, **nit** по усмотрению
- [ ] CI зелёный в GitHub Actions
- [ ] После зачёта: **merge** в `main` и **тег `PXX`**
